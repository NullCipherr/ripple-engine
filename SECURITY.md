# Política de Segurança

## Reporte de vulnerabilidades

Se você identificar uma vulnerabilidade de segurança, abra um reporte privado para os mantenedores do projeto via GitHub (Security Advisories), evitando exposição pública antes da análise.

## O que incluir no reporte

- Descrição do problema
- Vetor de ataque
- Impacto esperado
- Passos para reprodução
- Versões/ambiente afetado

## Boas práticas do projeto

- Não versionar segredos em arquivos de código.
- Manter dependências atualizadas.
- Revisar mudanças de build/publicação com atenção.
